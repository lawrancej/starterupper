Introduction: Learn git and project hosting jargon

Git is a version control system: it stores snapshots of a working directory over time in a repository. To take a snapshot, add changes to the staging area, then commit. A commit records the staging area with a message, timestamp, author, and links to parent commits. A commit's SHA1 hash identifies it and verifies integrity. Making a commit does not submit work.

Git history can split into separate timelines called branches, and merge back together. The default branch is master. Checkout a commit or a branch to see the working directory as it was (HEAD points at the checkout). During a merge, if two branches have different changes to the same places in a file, fix these conflicts manually or with a merge tool, then commit.

Git is distributed: every repository lists remote repositories, each with a nickname and URL. The default remote is origin. Project hosts such as Bitbucket, Github and Gitlab store and manage access to remote repositories. Push to submit work to a remote repository; pull to receive updates, or equivalently, fetch then merge.

Hints for using git
  • Commit often to keep commits small
  • Test first: don't push a broken master
  • Create branches for experiments
  • Push to origin, not to upstream
  • Fix CONFLICTs with a merge tool
Submit work from Terminal or Git Bash 
cd ~/REPOSITORY
git gui &
git add .
git commit -m "Message goes here"
git push --all origin
Receive updates with these steps 
cd ~/REPOSITORY
git commit -am "Message goes here"
git fetch --all
git merge upstream/master
git mergetool

Host setup: Setup an account and create a private repository for work submission

Join a project host with your school email address, then check your inbox. If you have an account already, add your school email address to it; do not join again. The project host will email a link that you must follow to verify that you own your email address. Be patient: email may take several minutes to arrive, and requesting a new verification email invalidates previous emails.

After verifying your email address, sign in to finish account setup and create a private repository shared with the instructor and nobody else. Note: Sign-in sends passwords securely to the project host, not the instructor.
