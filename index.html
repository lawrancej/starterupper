Starter Upper: Setup git and project hosting quickly

In this course, we'll use git repositories and project hosts (such as Github and Gitlab) to submit work, receive assignments and feedback. Git repositories store snapshots of a folder over time, enabling collaborators to work simultaneously along different time-lines called branches, which can be merged together. Git project hosts manage access to git repositories over the network.

Initially, your repository will be a copy of the course repository. To submit work, add files to git, record a snapshot as a commit, then push to your repository. The instructor will fetch and checkout your work. The instructor may also update the course repository; you may pull to receive those updates.

Hints for using git
  • Test and review git diff before you commit
  • Use branches for work-in-progress
  • Commit often, and to complete a merge
  • Do not push to upstream
  • Use git gui if you're confused
Submit work from the Terminal or Git Bash 
cd ~/REPOSITORY
git add .
git commit -m "Message goes here"
git push origin master
Receive updates following these steps 
cd ~/REPOSITORY
git commit -am "Message goes here"
git fetch --all
git merge upstream/master
git mergetool

Local setup: Setup git on your machine

The commands at right teach git your name, email address, and where to find the repositories we created. Git uses your name and email address to identify authors. Git also keeps a list of remote repositories in your local repository (see git remote -v), so that you can push your work to submit it and also receive updates doing a fetch then a merge, or by doing a pull). Every remote repository has a name and a URL:

  1. The course repository (upstream)
  2. Your private repository (bitbucket)
  3. Your private repository (github)
  4. Your private repository (gitlab)
  5. The local repository (~/REPOSITORY)
